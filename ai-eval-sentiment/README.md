# 感情分析 プロンプトA/B評価 — 試金石の実ドメイン適用

[試金石](../README.md)（競馬から抽出した定量検証スタック）を、LLMのプロンプト評価に適用した記録。題材＝感情分析のプロンプト A（素朴）vs B（皮肉・婉曲に注意せよと指示）。

## 問い
工夫したプロンプトBは、素朴なAより本当に良いのか？ まぐれか？ ——そして**その答えは、モデルを変えても保つのか？**

## Step 1（N=30）＝まだ測れない
| | 正解率 | 90%CI |
|---|---|---|
| A(素朴) | 90.0% | 80.0〜96.7% |
| B(工夫) | 96.7% | 90.0〜100.0% |
| **B−A** | **+6.7pp** | **+0.0〜+13.3pp（0をまたぐ）→ ⚠️まぐれと区別不能** |

観測上はBが良いが、N=30では「本物の差」と言えない。試金石は「証明には約110件要る」と提示（検出力）。

## Step 2（N=110・ただし分類はセッションのLLM自身＝手ラベル）＝手ラベル上は"本物"に見えた
| | 正解率 | 90%CI |
|---|---|---|
| A(素朴) | 90.0% | 85.5〜94.5% |
| B(工夫) | 97.3% | 94.5〜99.1% |
| **B−A** | **+7.3pp** | **+2.7〜+11.8pp（0を除外）→ ✅ 手ラベル上は頑健に良い（→Step 3で独立検証）** |

層別に割ると、効果は全部"難しい文"に居た（kind=clear +0.0pp / kind=tricky +22.9pp）。集計は効き所を隠す。

## Step 3（独立3モデル・実API）＝"改善"は消え、弱モデルでは逆効果
「分類器＝対話セッションのLLM自身」という内部妥当性の弱点を潰すため、`run_eval_realapi.py` で独立モデルに同じ110件×2プロンプトを実際に投げた：

| 分類器 | A(素朴) | B(工夫) | B−A | 90%CI | 判定 |
|---|---|---|---|---|---|
| Claude（セッション内・手ラベル/再現不能） | 90.0% | 97.3% | +7.3pp | +2.7〜+11.8 | ⚠️ "身内採点"＝結論の根拠には数えない |
| gemini-2.5-flash-lite | 99.1% | 99.1% | +0.0pp | −1.8〜+1.8 | 差なし（天井） |
| gpt-4o-mini | 95.5% | 97.3% | +1.8pp | −0.9〜+4.5 | ⚠️ ノイズと区別不能 |
| Llama-3.2-3b | 89.1% | 80.0% | **−9.1pp** | −16.4〜−1.8 | ❌ **頑健に悪い（逆効果）** |

**冒頭の+7.3ppは対話セッション内の手ラベル（＝この試金石が戒める"身内採点"）。それを疑って独立3モデルの実APIで測り直した結果が下の3行だ。** 頑健な改善はどのモデルにも無く、唯一はっきり出たのは弱モデル(Llama)での悪化。「プロンプト改善が効いた!」はモデル固有で、測った土俵の外には持ち出せない。分類器＝独立API（Claude行のみ手ラベル）。

## 再現方法
```bash
# 判定ロジックのみ（保存済みpredictionsで・APIキー不要）
python3 run_eval.py

# 実API・独立モデルで再測定（要 FAL_KEY / 数円〜数十円per model）
export FAL_KEY=<your-fal-api-key>
python3 run_eval_realapi.py google/gemini-2.5-flash-lite
python3 run_eval_realapi.py openai/gpt-4o-mini
python3 run_eval_realapi.py meta-llama/llama-3.2-3b-instruct
```
独立3モデルの生の測定結果は `predictions_real_*.json` として同梱（プロンプト全文も `_prompt_A`/`_prompt_B` に記載）。

## 限界（正直に）
- **Llamaの−9.1ppには分類失敗4件を含む**（指示した出力形式を守れず分類不能になったものを、不正解として数えた）。この4件を除くと**−6.6pp・90%CI[−13.2,+0.0]で上限が0に接する**＝"頑健な悪化"は全110件でのみ成立、失敗除外では有意ぎりぎりに弱まる。だから断定は"符号が反転しうる"まで。
- **Geminiの99.1%は天井効果**＝このテストセットは強モデルには易しすぎ、「差がない」と「検出できない」を区別できない。
- **テストセットは手作り110件**＝実運用の分布ではない。
- よって主張は個別数値の一般化ではなく**構造**：①点推定は嘘をつく（N=30の罠）②統計的に本物でも効果はモデル固有＝符号すら保存されない。

## ファイル
`testset.json`（110件・ラベル付き）／`predictions.json`（セッションLLM版・プロンプト全文入り）／`predictions_real_*.json`（独立3モデルの実測）／`run_eval.py`（判定）／`run_eval_realapi.py`（実API測定＋判定）
